Facilitator details

Description Details for a single facilitator
Request GET /api/v2/facilitator/{id}[?code={brandCode}]
Request path parameters identifier unique identifier for the person, e.g. 42 or sergey.kotlov
Query string parameters code — optional brand code, e.g. mgt30. If brand code is not used then the system returns only non-brand-related endorsements and materials, and year of experience and number of events always equal 0. If brand code is used then the system mixes non-brand-related materials with materials related to the given brand.
Response content type application/json; charset=utf-8
Response body

A JSON facilitator object, with properties:

  • id - a numeric unique identifier of a person, e.g. 42
  • unique_name - an unique identifier of a person, e.g. sergey.kotlov
  • first_name
  • last_name
  • email_address
  • image
  • address address object:
    • street1
    • street2
    • city
    • post_code
    • province
    • country - ISO 8166-2 country code
  • bio Markdown format
  • interests Markdown format
  • twitter_handle - Twitter user name, without the leading @@
  • facebook_url
  • linkedin_url
  • google_plus_url
  • website
  • blog
  • active
  • organizations - JSON array of organization objects, with the same structure as the organizations resource
  • licenses - JSON array of license objects, with properties:
    • brand - a brand code
    • start - date in `YYYY-MM-DD` format
    • end - date in `YYYY-MM-DD` format
  • endorsements - array of objects with properties:
    • content
    • name
    • company - contains either company or role
    • rating - if rating exists the endorsement is added by a participant
  • experiences - array of objects with properties:
    • type - possible values: article, casestudy, video
    • link
  • years_of_experience
  • number_of_events
  • rating
  • statistics - facilitator ratings, NPS, medians:
    • public_rating
    • private_rating
    • public_median
    • private_median
    • public_nps
    • private_nps
    • number_of_public_evaluations
    • number_of_private_evaluations
  • badges - array of badges with properties:
    • name
    • url
Errors (HTTP status)

404 Not Found if the specified person is not found
