Create

Description Adds new evaluation to the system in Unconfirmed state and sends a confirmation request to the participant
Request POST /api/v2/evaluation
Request POST parameters (deprecated)
  • event_id - unique identifier for the event, numeric
  • participant_id - unique identifier for the person who participated in the event, numeric
  • question1 - What is the reason you registered for this event?, string
  • question2 - Which action items did you take away from the course/event?, string
  • question3 - What can we add, change or delete in the content/courseware?, string
  • question4 - What were specific qualities of the facilitator(s)?, string
  • question5 - What can the host/organizer improve for the next event?, string
  • question6 - General impression of the trainer/facilitator?, numeric, [0,10]
  • question7 - How likely are you to recommend this event to others?, numeric, [0,10]
  • question8 - How can the facilitator(s) make the next event better?, string
Request POST parameters (new!)
  • event_id - unique identifier for the event, numeric
  • participant_id - unique identifier for the person who participated in the event, numeric
  • reason_to_register - What is the reason you registered for this event?, string
  • action_items - Which action items did you take away from the course/event?, string
  • changes_to_content - What can we add, change or delete in the content/courseware?, string
  • facilitator_review - What were specific qualities of the facilitator(s)?, string
  • changes_to_host - What can the host/organizer improve for the next event?, string
  • facilitator_impression - General impression of the trainer/facilitator?, numeric, [0,10]
  • recommendation_score - How likely are you to recommend this event to others?, numeric, [0,10]
  • changes_to_event - How can the facilitator(s) make the next event better?, string
  • content_impression - General impression of the content/courseware?, numeric, [0,10]
  • host_impression - General impression of the host/organizer?, numeric, [0,10]
Request query string parameters none
Response content type application/json; charset=utf-8
Response body

A JSON object with properties:

  • evaluationId - unique identifier for a created evaluation
Errors (HTTP status)

400 Bad Request if parameters are wrong, with a JSON object, containing errors for all parameters

Confirm

Description Confirms new evaluation and notifies the event facilitators and a brand owner
Request POST /api/v2/evaluation/:id/confirm
Request POST parameters none
Request query string parameters id - hashed string which uniquely identifies an evaluation in the system
Response content type application/json; charset=utf-8
Response body

A JSON object with properties:

  • success The evaluation is confirmed
Errors (HTTP status)

404 Not Found if the requests evaluation is not found
