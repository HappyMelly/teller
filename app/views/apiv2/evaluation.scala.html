Create

Description Adds new evaluation to the system in Unconfirmed state and sends a confirmation request to the participant
Request POST /api/v2/evaluation
Request POST parameters
  • eventId - unique identifier for the event, numeric
  • participantId - unique identifier for the person who participated in the event, numeric
  • question1 - What triggered you to register for this event?, string
  • question2 - Which action items did you take away from the event?, string
  • question3 - What do you suggest to add or change to the program?, string
  • question4 - What were specific qualities of the trainer/facilitator?, string
  • question5 - What can be improved about the event location?, string
  • question6 - General Impression, numeric, [0,10]
  • question7 - How likely are you to recommend this event to others?, numeric, [0,10]
  • question8 - How could the trainer/facilitator have made the event better?, string
Request query string parameters none
Response content type application/json; charset=utf-8
Response body

A JSON object with properties:

  • evaluationId - unique identifier for a created evaluation
Errors (HTTP status)

400 Bad Request if parameters are wrong, with a JSON object, containing errors for all parameters

Confirm

Description Confirms new evaluation and notifies the event facilitators and a brand owner
Request POST /api/v2/evaluation/:id/confirm
Request POST parameters none
Request query string parameters id - hashed string which uniquely identifies an evaluation in the system
Response content type application/json; charset=utf-8
Response body

A JSON object with properties:

  • success The evaluation is confirmed
Errors (HTTP status)

404 Not Found if the requests evaluation is not found
