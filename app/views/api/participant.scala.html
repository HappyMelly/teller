Participant creation

Description Add new participant to the system
Request POST /api/v1/participant
Request POST parameters (not member)
  • event_id unique identifier for the event, numeric
  • email string
  • first_name string
  • last_name string
  • birthday string, YYYY-mm-DD
  • organisation string
  • comment string
  • country - ISO 8166-2 country code, string
  • city string
  • street_1 string
  • street_2 string
  • province string
  • postcode string
Request POST parameters (member)
  • event_id unique identifier for the event, numeric
  • person_id unique identifier for the person, numeric
Request query string parameters none
Response content type application/json; charset=utf-8
Response body

A JSON object with properties:

  • participant_id - unique identifier for a created participant
Errors (HTTP status)

400 Bad Request if parameters are wrong, with a JSON object, containing errors for all parameters
